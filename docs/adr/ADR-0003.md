# ADR-0003: Pivot post-EPIC-3 — Entrada por DSL + Registry y Red Dirigida

## Estado
Aceptado

## Contexto
EPIC-3 entregó Elecboard IR v1 como modelo lógico estructural (grafo validado).
A partir de EPIC-4 se adopta un modelo de entrada basado en:
- DSL de Tableros (Board DSL)
- DSL de Red dirigida (Network DSL)
- Registry/DB como fuente de verdad de tags y componentes

## Decisión
Desde EPIC-4, la entrada canónica del sistema se declara mediante:
1) `boards/*.py`: tableros (cargas en el nivel más bajo)
2) `network/electrical_network.py`: vínculos dirigidos `supply_from(...).to(...).with_wire(...).done()`
3) Registry/DB: resolución de tags (board/columna/protección/bornera/cable)

El motor incorpora un Adapter que transforma estas entradas a una representación interna compatible
con el IR estructural existente (sin duplicación de cargas).

## Consecuencias
- La potencia se declara abajo y se agrega hacia arriba.
- El cable no contiene protección: solo referencia.
- La topología eléctrica se interpreta como grafo dirigido.
- Se habilita trazabilidad y automatización futura sin reescribir los tableros.
