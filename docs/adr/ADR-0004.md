# ADR-0004: Artifacts de Ingeniería como outputs reproducibles (Load/DAG)

## Estado
Aceptado

## Contexto
A partir de EPIC-04.01 y EPIC-04.02 el motor genera reportes de ingeniería derivados del DSL/Registry:
- Load aggregation (abajo → arriba)
- DAG dirigido (roots / reachability / cycles / unreachable)

Estos resultados deben quedar disponibles sin necesidad de re-ejecutar el motor y sin mutar el IR base.

## Decisión
Los resultados de ingeniería se materializan como:
1) Artifacts en filesystem:
   - `Reports/artifacts/load_report.json` + `load_report.md`
   - `Reports/artifacts/dag_report.json` + `dag_report.md`
2) Snapshot embebido en `payload.electrical_ir`:
   - `electrical_ir.load_report`
   - `electrical_ir.dag_report`
   - `electrical_ir.compile_report`
   - `electrical_ir.registry_snapshot`
   - `electrical_ir.artifacts` (paths)

## Consecuencias
- Reproducibilidad: artifacts regenerables desde mismas entradas.
- Auditabilidad: snapshot embebido y paths persistentes.
- Separación de capas: el IR estructural no se muta para “guardar cálculos”.
