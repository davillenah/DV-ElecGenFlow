# ADR-0001: Arquitectura elecgenflow

## Estado
Aceptado

## Decisión
Se adopta Domain-Driven Design (DDD) y el patrón Model-View-Presenter (MVP).
El motor mantiene separación estricta entre:
- Modelo/contratos (Domain)
- Orquestación (Engine)
- Servicios de ingeniería (Load, DAG, Sizing, Normas)
- Simulación (pandapower) y evaluación

## Consecuencias
- El modelo lógico (IR) es estable y validable.
- Los cálculos, normas y selección de componentes se implementan como servicios desacoplados.
- Se preserva reproducibilidad (seed + manifest).
