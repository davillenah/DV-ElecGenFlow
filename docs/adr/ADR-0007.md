# ADR-0007: PDF precursor generado exclusivamente desde artifacts (EPIC-11 precursor)

## Estado
Aceptado

## Contexto
El motor ya genera artifacts reproducibles en `Reports/artifacts/`:
- `load_report.json/.md` (EPIC-04.01)
- `dag_report.json/.md` (EPIC-04.02)
- `nominal_snapshot.json/.md` y `nominal_overlay_diff.json/.md` (EPIC-04.03)

Se requiere un primer PDF técnico consolidado para revisión y entrega, sin implementar aún el motor final de reportes (EPIC-11 completo).

## Decisión
Se implementa un generador PDF “precursor” que:
1) Lee artifacts `.json`/`.md` existentes desde `Reports/artifacts/`.
2) Renderiza un PDF consolidado con:
   - Portada KPI (resumen ejecutivo)
   - Secciones de reportes por artefacto
   - Estilo consistente (header/footer, tablas y tipografías)
3) Produce el archivo:
   - `Reports/artifacts/engineering_report.pdf`
4) No recalcula magnitudes ni muta el IR; solo consume artifacts ya generados.

## Consecuencias
- Se habilita un PDF reproducible y auditable sin introducir estado nuevo.
- EPIC-11 sigue pendiente como “motor final”, pero queda un paso intermedio útil para ingeniería.
- La calidad del PDF mejora incrementalmente sin romper el pipeline existente.
