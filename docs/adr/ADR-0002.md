# ADR-0002: Reordenamiento del Roadmap por Dependencia de Capas Lógicas

## Estado
Aceptado

## Decisión
Se establece la reestructuración obligatoria del roadmap de desarrollo, priorizando la integridad del modelo de datos sobre la ejecución de procesos de cálculo. Se dispone el posicionamiento de la **Librería de Componentes (EPIC-4)** como requisito previo a la construcción del modelo eléctrico real. Asimismo, se determina el aplazamiento consciente del motor de simulación y el sistema de reglas hasta que se garantice la existencia de una base de datos técnica sólida. Se incorporan, de manera mandatoria, capas intermedias de gestión de escenarios, calidad de potencia y coordinación de protecciones antes del lanzamiento de la versión de producción.

## Consecuencias
Como resultado de esta directriz, se asegura que el motor de simulación (`pandapower`) y el motor de reglas operen exclusivamente sobre entidades con parámetros físicos validados, eliminando la carga manual de datos inconsistentes. Se establece una arquitectura donde la lógica de negocio se apoya en un modelo de información robusto, lo que reduce la necesidad de refactorizaciones futuras en el núcleo del dominio. Finalmente, se garantiza una trazabilidad superior en la comparación de variantes y se eleva el estándar de seguridad técnica requerido para la validación industrial en la versión v1.0.0.
