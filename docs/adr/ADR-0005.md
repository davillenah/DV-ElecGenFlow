# ADR-0005: Normalización de unidades + PF configurable (sin asumir datos críticos)

## Estado
Aceptado

## Contexto
En proyectos reales las cargas pueden declararse en:
VA, kVA, MVA, W, kW, MW, HP.
Además, para convertir entre potencia aparente y activa se requiere PF.

En esta etapa no existe aún un modelo eléctrico completo (EPIC-05), pero se necesita consistencia para reportes.

## Decisión
1) Se normaliza toda carga a (kW, kVA) en reportes de ingeniería.
2) PF se toma como valor por defecto configurable:
   - `configs/default_ar.yaml: power_factor_default`
3) Regla de conversión (por ahora):
   - si viene kVA → kW = kVA * PF
   - si viene kW → kVA = kW / PF
   - HP → kW (1 HP = 0.746 kW) y luego kVA = kW / PF

## Consecuencias
- Reportes coherentes y comparables (kW/kVA).
- No se “inventa” PF: queda declarado en config.
- Más adelante EPIC-05 reemplazará/expandirá estas conversiones con modelo P/Q real.
