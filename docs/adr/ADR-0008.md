# ADR-0008: Overlays nominales segmentados por fabricante y auto-discovery no bloqueante

## Estado
Aceptado

## Contexto
Las tablas nominales (EPIC-04.03) requieren:
- Base versionada (`data/nominal/v0/*.json`)
- Overlays por fabricante/familia/modelo

Se necesita permitir que el usuario agregue nuevos fabricantes sin tocar configuración y sin romper la ejecución si hay JSON incompletos.

## Decisión
1) Se adopta estructura segmentada para overlays:
   - `data/nominal/overlays/cables/<fabricante>/cables.json`
   - `data/nominal/overlays/protections/<fabricante>/<familia>/protections.json`
2) Se habilita auto-discovery:
   - Si `nominal_overlays` está vacío, se escanean todos los `*.json` bajo `data/nominal/overlays/**`.
3) Comportamiento no bloqueante:
   - Si un overlay no tiene `overlay=true`, se ignora y se registra warning.
   - Solo overlays marcados con `overlay=true` se aplican.

## Consecuencias
- Se pueden agregar fabricantes (ABB, Siemens, Schneider, etc.) sin editar config.
- El pipeline no se cae por archivos intermedios o borradores.
- El “diff” de overlays refleja únicamente overlays aplicados (overlay=true).
